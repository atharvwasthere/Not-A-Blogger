# Not a Blogger

> An engineering journal by Atharv Singh.

I'm a backend engineer, and this is where I write about the things I actually think about at work — system design, debugging sessions that took way too long, architecture decisions, and the reasoning behind them.

It's not a tutorial blog. It's more like a thinking log that happens to be public.

---

## What's in here

- Backend architecture breakdowns
- Real debugging case studies
- Tradeoff analysis I've done or wish I'd done earlier
- Production failure post-mortems
- Performance investigations
- Engineering decision logs

---

## Why "Not a Blogger"

Because I'm not one.

I'm not chasing trends, writing beginner guides, or optimizing for shares. I write when I have something worth writing — something I'd want to read myself, or send to a colleague who's stuck on the same problem.

Every post is closer to an engineering memo than a blog post. Depth over cadence. Signal over noise.

---

## Tech Stack

**Frontend** — React, TanStack Router / Start, Tailwind CSS

**Backend** — Python / FastAPI, Postgres, SQLAlchemy

**Infrastructure** — Docker for local dev, Azure App Service, GitHub Actions for CI/CD

I care about fast load times, clean URLs, and proper structured data — not because I'm obsessed with SEO, but because a site that's slow and poorly structured is just bad engineering.

Why would you want to run this locally? I don't know, you tell me.

## Deployment

Runs on Azure App Service, containerized with Docker, pointed at a managed Postgres instance. Config is environment-based — no surprises in prod.
